网络通信的例子

两台计算机之间通过网线连接可以传递文件，但是要手动配置两台计算机在同一网段
多台计算机接在一个路由器下面，同为一个局域网
计算机下载文件必须要连接到Internet

信息传递过程

局域网、城域网、广域网


局域网是连接同一路由器的网络（个人热点也是局域网）
公网是广域网使用的ip地址，可以有无数个私网地址，但是同一局域网不要重复

IP地址的表示

ipv4和ipv6的区别


ipv6应用路由器，交换机，ofo等

公网地址和私网地址


私有网络连接到Internet是使用nat技术实现的

私网ip地址

内网地址通过nat技术转换成公网地址来连接到Internet上网

特殊IP地址


网段

A段：前8位，B段：第9位到第16位，以此类推

扫秒C段

C段定位，C段不变，实际上是扫D段，可以知道企业买了几个连号，比如百度http://110.242.68.10/,3-12几乎都是百度搜索

近源攻击

近源攻击是指攻击者通过物理接触或者局域网内渗透等方式，直接接触目标系统或网络，进行攻击的一种方式。

虚拟机的nat模式和桥接模式

虚拟机能上网，用nat模式，nat模式是连接到物理机，将物理机当做路由器进行数据转发
桥接不要选择自动，无线网卡选wifi6（或许不一样），有线网卡选自己的有线网卡。
桥接和自己物理机平起平坐，也会获取一个真实ip，和物理机在同一局域网。（物理机必须连wifi不然访问不到虚拟机）
别人连自己的虚拟机用桥接

HFS

文件共享，一个上传文件的软件

1.首先将虚拟网络编辑器打开，加入VMnet0，设置成桥接模式，桥接至

2.点击确定，右击win10虚拟机设置桥接模式，勾选复制物理网络连接复制状态，

3.将hfs.exe拖入win10虚拟机，新建一个txt文档拖入hfs.exe，

就可以在连接了同一wifi的局域网内访问虚拟机的这个文档


端口号：1-65535，一个端口对应一个服务

nmap使用

（功能：指纹识别，端口扫描，漏洞挖掘）
（指纹：推断目标系统的具体类型和配置，分析网站使用的什么语言，插件等，可以在这上面找漏洞）

1.打开kali，nmap -h打开使用指南

2.扫域名的ip地址（这里扫的是百度）

配置ssh

cd ~/.ssh #检查本机已存在的ssh密钥

  • 如果提示：No such file or directory 说明你是第一次使用git。

ssh-keygen -t rsa -C “邮件地址”（生成ssh密钥对，密钥类型rsa，-C跟的是注释字段）

  • 然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

  • key填写C:\Users\Administrator.ssh\id_rsa.pub中的内容
  • 尝试ssh连接,测试是否成功

ssh -T git@github.com # 注意邮箱地址不用改

  • 如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到下面这个，就算成功：

Hi liuxianan! You’ve successfully authenticated, but GitHub does not provide shell access.

  • 在选中的位置(blog目录下)打开cmd(以管理员身份)

npm install hexo-deployer-git –save（将Hexo博客部署到Git仓库）

  • 在上一个Git Bash Here还需要继续配置

git config –global user.name “luozhixiaowo”// 你的github用户名，非昵称
git config –global user.email  “xxx@qq.com“// 填写你的github注册邮箱
